<h2><a href="https://www.postgresql.org/docs/14/sql-truncate.html" target="_blank"><code>TRUNCATE TABLE</code></a></h2 ><p>O PostgreSQL possui o comando <code>TRUNCATE</code> que permite remover dados (linhas) de uma tabela sem remover sua estrutura.</p><p> </p><p>Temos disse anteriormente. que para descartar uma tabela usaremos o comando <code>DROP TABLE</code>.</p><h3>Sintaxe:</h3><blockquote><pre><code>TRUNCATE [TABLE] TableName [CASCADE] ;</code></pre></blockquote><p>A opção <code>CASCADE</code> pode ser incluída nos casos em que uma das colunas é uma chave estrangeira em outra tabela filha, o que impediria sua exclusão . Colocando esta opção, o PostgreSQL trunca todas as tabelas filhas que se referem à tabela pai com uma restrição de integridade referencial do tipo <code>ON DELETE CASCADE</code>.</p><h3>Exemplo:</h3><p >Vamos imagine que criamos a tabela <code>USERS</code> e inserimos vários registros:</p><blockquote><p><code>CREATE TABLE USERS (<br /> Login VARCHAR(15) PRIMARY KEY , <br /> Senha VARCHAR(8) NOT NULL,<br /> HireDate DATE DEFAULT CURRENT_DATE<br />);</code></p><p><code>INSERT INTO USERS(Login, Password) VALUES ( 'User1', '12345678');<br />INSERT INTO USERS(Login, Password) VALUES('User2', '23456789');<br />INSERT INTO USERS(Login, Password) VALUES('User3' , '34567890');</code></p></blockquote><p>Se executarmos posteriormente o comando</p><blockquote><pre><code>TRUNCATE TABLE USERS;</code></ pre></blockquote><p>Nós excluiríamos o conteúdo. Portanto, a resposta a um </p><blockquote><p><code>SELECT * FROM USERS;</code></p></blockquote><p>retornaria </p><blockquote><pre> <código>entrar | senha | contratado<br />-------+----------+----------<br />(0 linhas)</code></pre> </blockquote><p>Mas, em vez disso, a estrutura da tabela <code>USERS</code> ainda apareceria com uma consulta como:</p><blockquote><pre><code>SELECT table_name, column_name, data_type <br / >FROM information_schema.columns<br />WHERE table_name = 'users';</code></pre></blockquote><h3>Exercício:</h3><p>Imagine que, nos dados base, temos a tabela a seguir, com seus respectivos dados:</p><blockquote><p><code>CREATE TABLE MODULES (<br /> CODE INTEGER PRIMARY KEY,<br /> NAME VARCHAR(100),< br /> DEPARTMENT CHAR(3)<br />);<br /> <br />INSERT INTO MODULES VALUES (1, 'PROGRAMMING', 'PRG');<br />INSERT INTO MODULES VALUES (2, 'DATABASES', ' DBS');</code></p><p> </p></blockquote><p>Apaga os registros da tabela, sem deletar sua estrutura.</p>
