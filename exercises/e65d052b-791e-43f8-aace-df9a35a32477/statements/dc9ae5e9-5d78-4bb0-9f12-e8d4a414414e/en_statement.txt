<h2><a href="https://www.postgresql.org/docs/14/sql-truncate.html" target="_blank"><code>TRUNCATE TABLE</code></a></h2 ><p>PostgreSQL has the <code>TRUNCATE</code> command that will allow you to remove data (rows) from a table without removing its structure.</p><p> </p><p>We have said previously. that to drop a table we will use the command <code>DROP TABLE</code>.</p><h3>Syntax:</h3><blockquote><pre><code>TRUNCATE [TABLE] TableName [CASCADE];< /code></pre></blockquote><p>The <code>CASCADE</code> option can be included for cases where one of the columns is a foreign key in another child table, which would prevent its deletion. Placing this option PostgreSQL truncates all child tables that refer to the parent table with a referential integrity constraint of the type <code>ON DELETE CASCADE</code>.</p><h3>Example:</h3><p >Let's imagine that we have created the table <code>USERS</code> and we have inserted several records:</p><blockquote><p><code>CREATE TABLE USERS (<br /> Login VARCHAR(15) PRIMARY KEY, <br /> Password VARCHAR(8) NOT NULL,<br /> HireDate DATE DEFAULT CURRENT_DATE<br />);</code></p><p><code>INSERT INTO USERS(Login, Password) VALUES( 'User1', '12345678');<br />INSERT INTO USERS(Login, Password) VALUES('User2', '23456789');<br />INSERT INTO USERS(Login, Password) VALUES('User3', '34567890');</code></p></blockquote><p>If we subsequently execute the command</p><blockquote><pre><code>TRUNCATE TABLE USERS;</code></ pre></blockquote><p>We would delete the content. So the response to a </p><blockquote><p><code>SELECT * FROM USERS;</code></p></blockquote><p>would return </p><blockquote><pre> <code>login | password | hiredate<br />-------+----------+----------<br />(0 rows)</code></pre> </blockquote><p>But instead the <code>USERS</code> table structure would still appear with a query like:</p><blockquote><pre><code>SELECT table_name, column_name, data_type <br />FROM information_schema.columns<br />WHERE table_name = 'users';</code></pre></blockquote><h3>Exercise:</h3><p>Imagine that, in the base data, we have the following table, with its respective data:</p><blockquote><p><code>CREATE TABLE MODULES (<br /> CODE INTEGER PRIMARY KEY,<br /> NAME VARCHAR(100),< br /> DEPARTMENT CHAR(3)<br />);<br /> <br />INSERT INTO MODULES VALUES (1, 'PROGRAMMING', 'PRG');<br />INSERT INTO MODULES VALUES (2, ' DATABASES', 'DBS');</code></p><p> </p></blockquote><p>Deletes the records of the table, without deleting its structure.</p>
