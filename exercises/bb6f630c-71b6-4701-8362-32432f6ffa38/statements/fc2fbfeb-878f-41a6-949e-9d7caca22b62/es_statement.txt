<h2>Crear &iacute;ndices.</h2><p>Sabemos que crear &iacute;ndices ayuda a la localizaci&oacute;n m&aacute;s r&aacute;pida de la informaci&oacute;n contenida en las tablas.</p><h3>Sintaxis:</h3><blockquote><pre><code>CREATE INDEX [IndexName] ON TableName (Column1 [, Column2 ...]);</code></pre></blockquote><p>No es aconsejable que utilices campos de tablas peque&ntilde;as o que se actualicen con mucha frecuencia. Tampoco es conveniente si esos campos no se usan en consultas de manera frecuente o en expresiones.</p><p>La mayor&iacute;a de los &iacute;ndices se crean de manera impl&iacute;cita cuando ponemos las restricciones <code>PRIMARY KEY</code>, <code>FOREIGN KEY</code> o <code>UNIQUE</code>.</p><h3>Ejemplo:</h3><p>Tenemos creada la siguiente tabla:</p><blockquote><pre><code>CREATE TABLE MY_EMPLOYEES (<br />    COD_EMPLOYEE VARCHAR(5) PRIMARY KEY,<br />    FIRST_NAME VARCHAR(15),<br />    LAST_NAME VARCHAR(30),<br />    SALARY NUMERIC(6,2)<br />);</code></pre></blockquote><p>Crea un &iacute;ndice con el campo <code>LAST_NAME</code>.</p><blockquote><pre><code>CREATE INDEX employeesLastNameIndex ON MY_EMPLOYEES (LAST_NAME);</code></pre></blockquote><h3>Ejercicio:</h3><p>Imagina que hemos creado la siguiente tabla:</p><blockquote><pre><code>CREATE TABLE MODULES (<br />&nbsp;&nbsp;&nbsp; Name VARCHAR(100),<br />&nbsp;&nbsp;&nbsp; NumHours INTEGER,<br />&nbsp;&nbsp;&nbsp; Code NUMERIC(4),<br />&nbsp;&nbsp;&nbsp; Skill VARCHAR(100)<br />);</code></pre></blockquote><p>Crea un &iacute;ndice con el atributo <em>Name</em>.</p>
