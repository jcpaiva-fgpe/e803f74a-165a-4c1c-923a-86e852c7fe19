<h2>Create indexes.</h2><p>We know that creating indexes helps to locate the information contained in the tables faster.</p><h3>Syntax:</h3><blockquote><pre> <code>CREATE INDEX [IndexName] ON TableName (Column1 [, Column2 ...]);</code></pre></blockquote><p>It is not recommended that you use small table fields or update them with very often. It's also not convenient if those fields aren't used in frequent queries or expressions.</p><p>Most indexes are created implicitly when we put the <code>PRIMARY KEY</code> constraints, < code>FOREIGN KEY</code> or <code>UNIQUE</code>.</p><h3>Example:</h3><p>We have created the following table:</p><blockquote><pre> <code>CREATE TABLE MY_EMPLOYEES (<br /> COD_EMPLOYEE VARCHAR(5) PRIMARY KEY,<br /> FIRST_NAME VARCHAR(15),<br /> LAST_NAME VARCHAR(30),<br /> SALARY NUMERIC(6,2) <br />);</code></pre></blockquote><p>Create an index with the <code>LAST_NAME</code> field.</p><blockquote><pre><code>CREATE INDEX employeesLastNameIndex ON MY_EMPLOYEES (LAST_NAME);</code></pre></blockquote><h3>Exercise:</h3><p>Imagine that we have created the following table:</p><blockquote><pre> <code>CREATE TABLE MODULES (<br /> Name VARCHAR(100),<br /> NumHours INTEGER,<br /> Code NUMERIC(4),<br /> Skill VARCHAR(100)<br />);< /code></pre></blockquote><p>Create an index with the <em>Name</em> attribute.</p>
