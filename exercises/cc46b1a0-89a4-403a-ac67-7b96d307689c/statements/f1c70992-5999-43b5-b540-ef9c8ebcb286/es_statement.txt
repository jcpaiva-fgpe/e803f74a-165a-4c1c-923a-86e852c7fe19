<p>Es posible que, despu&eacute;s de crear una tabla, nos demos cuenta que se nos ha olvidado a&ntilde;adir alg&uacute;n campo o restricci&oacute;n, quiz&aacute;s alguna de las restricciones que a&ntilde;adimos ya no es necesaria o tal vez queramos cambiar el nombre de alguno de los campos. &iquest;Es posible esto? Ahora veremos que s&iacute; y en casi todos los casos utilizaremos el comando <a href="https://www.postgresql.org/docs/14/sql-altertable.html" target="_blank"><code>ALTER TABLE</code></a>.</p><h2>Renombrar tablas</h2><p>Si queremos cambiar el nombre de una tabla utilizaremos la cl&aacute;usula <code>RENAME</code> del comando <code>ALTER TABLE</code>.</p><h3>Sintaxis:</h3><blockquote><pre><code>ALTER TABLE [ IF EXISTS ] TableName<br />    RENAME TO new_name;</code></pre></blockquote><h3>Ejemplo:</h3><p>En un ejercicio anterior hemos creado la tabla <code>USERS</code> con la siguiente sentencia:</p><blockquote><pre><code>CREATE TABLE USERS (<br />    FIRST_NAME VARCHAR(25),<br />    LAST_NAME VARCHAR(30)<br />);</code></pre></blockquote><p>Para renombrar la tabla como <code>STUDENTS</code>, deber&iacute;amos ejecutar el siguiente comando:</p><blockquote><p><code>ALTER TABLE <strong>USERS</strong> RENAME TO <strong>STUDENTS</strong>;</code></p></blockquote><h3>Ejercicio:</h3><p>Imagina que hemos creado la siguiente tabla:</p><blockquote><pre><code>CREATE TABLE SUBJECTS (<br />    Name VARCHAR(100),<br />    NumHours INTEGER<br />);</code></pre></blockquote><p>Cambia el nombre de la tabla <code>SUBJECTS</code> por el de <code>MODULES</code>.</p>
