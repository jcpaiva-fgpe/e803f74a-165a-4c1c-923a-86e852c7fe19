<p>É possível que, após criar uma tabela, percebamos que nos esquecemos de adicionar algum campo ou restrição, talvez algumas das restrições que adicionamos não sejam mais necessárias ou talvez queiramos alterar o nome de algumas das campos. Isso é possível? Agora veremos que é e em quase todos os casos usaremos o comando <a href="https://www.postgresql.org/docs/14/sql-altertable.html" target="_blank"><code >ALTER TABLE< /code></a>.</p><h2>Renomear tabelas</h2><p>Se quisermos alterar o nome de uma tabela, usaremos o <code>RENAME</code> cláusula do <code>comando ALTER TABLE </code>.</p><h3>Sintaxe:</h3><blockquote><pre><code>ALTER TABLE [ IF EXISTS ] TableName<br /> RENAME TO new_name ;</code></pre> </blockquote><h3>Exemplo:</h3><p>Em um exercício anterior criamos a tabela <code>USERS</code> com a seguinte instrução:</p> <blockquote><pre><code> CREATE TABLE USERS (<br /> FIRST_NAME VARCHAR(25),<br /> LAST_NAME VARCHAR(30)<br />);</code></pre></blockquote> <p>Para renomear a tabela como < code>STUDENTS</code>, devemos executar o seguinte comando:</p><blockquote><p><code>ALTER TABLE <strong>USERS</strong> RENAME TO < strong>ESTUDANTES</strong>;</ code></p></blockquote><h3>Exercício:</h3><p>Imagine que criamos o seguinte Próxima tabela:</p><blockquote><pre><code>CREATE TABLE SUBJECTS (<br /> Nome VARCHAR(100),<br /> NumHours INTEGER<br />);</code></pre> </blockquote><p>Renomeie a tabela <code>SUBJECTS</code> para <code>MODULES</code>.</p>
