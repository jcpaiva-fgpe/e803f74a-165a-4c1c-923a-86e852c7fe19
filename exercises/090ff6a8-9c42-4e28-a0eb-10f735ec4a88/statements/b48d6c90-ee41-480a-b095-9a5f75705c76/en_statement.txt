<h2>Modify a table. Table constraints.</h2><p>We can also add table constraints, required when the constraint affects more than one column. For example, when the primary key or foreign key is made up of more than one attribute.</p><h3>Syntax:</h3><blockquote><p><code>ALTER TABLE TableName ADD <em>table_constraint< /em>;</code></p></blockquote><p>Where <em>table_constraint</em> could be of type <code>UNIQUE</code>, <code>PRIMARY KEY</code> , <code>FOREIGN KEY</code> or <code>CHECK</code>.</p><h3>Example:</h3><p>Imagine that we have executed the following table creation statement:</ p><blockquote><pre><code>CREATE TABLE STUDENTS (<br /> LAST_NAME VARCHAR(50) NOT NULL,<br /> FIRST_NAME VARCHAR(25) NOT NULL,<br /> BIRTH_DATE DATE,<br /> COURSE INTEGER<br />);</code></pre></blockquote><p>But, it is necessary to incorporate, as primary key, the set of attributes <code>LAST_NAME</code>, <code> FIRST_NAME</code> and <code>BIRTH_DATE</code>:</p><blockquote><code>ALTER TABLE STUDENTS<br /> ADD PRIMARY KEY (LAST_NAME, FIRST_NAME, BIRTH_DATE);</code></blockquote ><h3>Exercise:</h3><p>Imagine that we have created or the following tables:</p><blockquote><p><code>CREATE TABLE MODULES (<br /> CODE INTEGER,<br /> NAME VARCHAR(100),<br /> DEGREE CHAR(3)<br />);</code></p><p><code>CREATE TABLE BOOKS (<br /> ISBN NUMERIC,<br /> TITLE VARCHAR(100),<br /> MODULE INTEGER,</code> <br /><code> DEGREE CHAR(3)<br />);</code></p></blockquote><p>Incorporates the following restrictions:</p><ul><li>The key primary of table <code>MODULES</code> must be the set of <code>CODE</code> and <code>DEGREE</code>.</li><li>The primary key of table <code >BOOKS</code> must be the <code>ISBN</code> attribute.</li><li>The set (<code>MODULE</code>, <code>DEGREE</code>) of the table <code>BOOKS</code> references the <code>MODULES</code> table.</li></ul><p> </p>
