<h2>Restrição <code>CHAVE PRIMÁRIA</code> II.</h2><p>Só pode haver uma chave primária por tabela, mas pode ser composta por várias colunas.</p><p>Se a chave é composta por mais de uma coluna, por exemplo <code>FIRST_NAME</code>, <code>LAST_NAME</code>, <code>BIRTH_DATE</code>:</p><blockquote><pre> <code> CREATE TABLE USERS (<br /> FIRST_NAME VARCHAR(25),<br /> LAST_NAME VARCHAR(30),<br /> BIRTH_DATE DATE,<br /> CONSTRAINT Users_PK PRIMARY KEY(FIRST_NAME, LAST_NAME, BIRTH_DATE)< br /> );</code></pre></blockquote><h3>Exercício:</h3><p>A seguir está uma tabela que armazena as informações dos <code>MÓDULOS</code> em que os alunos podem ser matriculados:</p><blockquote><pre><code>CREATE TABLE MODULES (<br /> CODE INTEGER NOT NULL,<br /> NAME VARCHAR(100),<br /> DEGREE CHAR(3) ) <br /> );</code></pre></blockquote><p>Existem <code>MÓDULOS</code> que, tendo o mesmo <code>CÓDIGO</code>, podem ser ensinados em diferentes < code>DEGREE</code>, como Bancos de Dados. Portanto, na referida tabela não há atributo que, por si só, possa identificar cada um dos registros. Nesse caso, devemos identificar a chave primária como composta pelos atributos <code>CODE</code> e <code>DEGREE</code>.</p>
