<p>Los objetos b&aacute;sicos con los que trabaja <code>SQL</code> para almacenar los datos son las <strong>tablas</strong>
, que como ya sabemos son un conjunto de <em>filas</em> y <em>columnas</em> cuya intersecci&oacute;n se llama <em>celda</em>. Es ah&iacute; donde se almacenar&aacute;n los elementos de informaci&oacute;n, los datos que queremos recoger.</p><p>Antes de crear la tabla es conveniente planificar algunos detalles:</p><ul><li><strong>Qu&eacute; nombre</strong> le vamos a dar <strong>a la tabla</strong>.</li><li><strong>Qu&eacute; nombre</strong> le vamos a dar a cada una de las <strong>columnas</strong>.</li><li><strong>Qu&eacute; tipo y tama&ntilde;o de datos</strong> vamos a almacenar en cada columna.</li><li><strong>Qu&eacute; restricciones</strong> tenemos sobre los datos.</li><li>Alguna otra <strong>informaci&oacute;n adicional</strong> que necesitemos.</li></ul><p>Y debemos tener en cuenta otras <strong>reglas</strong> que se deben cumplir <strong>para los nombres de las tablas</strong>:</p><ul><li>No podemos tener nombres de tablas duplicados en un mismo esquema (usuario).</li><li>Deben comenzar por un car&aacute;cter alfab&eacute;tico.</li><li>Su longitud m&aacute;xima es de 30 caracteres.</li><li>Solo se permiten letras del alfabeto ingl&eacute;s, d&iacute;gitos o el signo de gui&oacute;n bajo.</li><li>No puede coincidir con las palabras reservadas de <code>SQL</code> (por ejemplo, no podemos llamar a una tabla <code>WHERE</code>).</li><li>No se distingue entre may&uacute;sculas y min&uacute;sculas.</li><li>En el caso de que el nombre tenga espacios en blanco o caracteres nacionales (permitido s&oacute;lo en algunas bases de datos), entonces se suele entrecomillar con comillas dobles.</li></ul><h3>Sintaxis</h3><p>La sintaxis b&aacute;sica del comando que permite crear una tabla es la siguiente:</p>
<blockquote><pre><code>CREATE TABLE [schema.]tablename (
    column1 datatype,
    column2 datatype,
    ...
    columnN datatype
);</code></pre></blockquote>
<p>donde:</p><ul><li><code>column1, column2, ..., columnN</code> son los nombres de las columnas que contendr&aacute; la tabla.</li><li><code>datatype</code> indica el tipo de dato de cada columna.</li></ul><h3>Ejemplo:</h3><p>Para crear una tabla llamada <strong>STUDENTS</strong> con dos campos de tipo <code>VARCHAR</code>:</p>
<blockquote><pre><code>CREATE TABLE STUDENTS (
    firstName VARCHAR(25),
    lastName VARCHAR(50)
);</code></pre></blockquote>

No obstante, como seguro que vas a tener que borrar tablas para poder rehacerlas correctamente o con mayor precisión, deberás utilizar el comando <code>DROP TABLE table_name;</code>. Por ejemplo: <ul><li><code>DROP TABLE USERS;</code>,</li><li><code>DROP TABLE MODULES;</code> o</li><li><code>DROP TABLE STUDENTS;</code></small></li></ul></p>
