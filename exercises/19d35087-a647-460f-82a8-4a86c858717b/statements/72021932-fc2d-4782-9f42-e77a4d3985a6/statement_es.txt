<h2>Restricci&oacute;n <a href="https://www.postgresql.org/docs/14/ddl-constraints.html#DDL-CONSTRAINTS-CHECK-CONSTRAINTS" target="_blank"><code>CHECK</code></a>.</h2><p>Tambi&eacute;n vamos a necesitar que se compruebe que los valores que se introducen en una columna determinada son admisibles. Para ello utilizaremos <code>CHECK</code>.</p><p>Esta restricci&oacute;n comprueba que se cumpla una condici&oacute;n determinada al rellenar una columna. Dicha condici&oacute;n se puede construir con columnas de esa misma tabla.</p><p>Si en la tabla USERS tenemos el campo <code>CREDIT</code> y &eacute;ste s&oacute;lo puede estar entre 0 y 2000, lo especificar&iacute;amos as&iacute;:</p><blockquote><pre><code>CREATE TABLE USERS (<br />    CREDIT INTEGER CHECK (CREDIT BETWEEN 0 AND 2000)<br />);</code></pre></blockquote><p>Una misma columna puede tener varios <code>CHECK</code> asociados a ella, para ello ponemos varios <code>CONSTRAINT</code> seguidos y separados por comas.</p><h3>Ejercicio:</h3><p>Debemos asegurarnos de que el <code>EMAIL</code> de los estudiantes contenga, al menos, una <code>@</code>, para lo que necesitaremos a&ntilde;adir una restricci&oacute;n <code>CHECK</code> que utilice la funci&oacute;n <a href="https://www.postgresql.org/docs/14/functions-string.html" target="_blank"><code>POSITION</code></a> para encontrar una <code>'@'</code> en la columna <code>EMAIL</code>.</p>
