<h2>Restrição <a href="https://www.postgresql.org/docs/14/ddl-constraints.html#DDL-CONSTRAINTS-CHECK-CONSTRAINTS" target="_blank"><code>CHECK</code ></a>.</h2><p>Também precisaremos verificar se os valores inseridos em uma determinada coluna são admissíveis. Para isso usaremos <code>CHECK</code>.</p><p>Esta restrição verifica se uma determinada condição é atendida ao preencher uma coluna. Esta condição pode ser construída com colunas dessa mesma tabela.</p><p>Se na tabela USERS temos o campo <code>CREDIT</code> e só pode estar entre 0 e 2000, nós o especificamos assim:</p><blockquote><pre><code>CREATE TABLE USERS (<br /> CREDIT INTEGER CHECK (CREDIT BETWEEN 0 AND 2000)<br />);</code></pre></ blockquote><p >A mesma coluna pode ter vários <code>CHECK</code> associados a ela, para isso colocamos vários <code>CONSTRAINT</code> seguidos e separados por vírgulas.</p><h3>Exercício :</h3> <p>Precisamos ter certeza de que o <code>EMAIL</code> dos alunos contém pelo menos um <code>@</code>, para o qual precisaremos adicionar um <code>CHECK </code> restrição que usa a função <a href="https://www.postgresql.org/docs/14/functions-string.html" target="_blank"><code>POSITION</code></ a> para encontrar um < code>'@'</code> na coluna <code>EMAIL</code>.</p>
