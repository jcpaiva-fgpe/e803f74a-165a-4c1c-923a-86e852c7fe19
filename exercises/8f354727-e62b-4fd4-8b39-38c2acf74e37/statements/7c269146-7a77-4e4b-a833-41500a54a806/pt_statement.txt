<p>Já dissemos antes. que para descartar uma tabela usaremos o comando <code>DROP TABLE</code>.</p><h3>Sintaxe:</h3><blockquote><pre><code>DROP TABLE TableName [CASCADE];< /code ></pre></blockquote><p>A opção <code>CASCADE</code> pode ser incluída nos casos em que uma das colunas é uma chave estrangeira em outra tabela filha, o que impediria sua exclusão. Ao colocar esta opção as restrições onde está a chave estrangeira serão apagadas antes e então a tabela em questão será apagada.</p><h3>Exemplo:</h3><p>Imagine que criamos duas tabelas relacionadas, como <code> MATCHES</code> e <code>USERS</code>, com as seguintes instruções:</p><blockquote><pre><code>CREATE TABLE MATCHES (<br /> COD_MATCH INTEGER,<br / > MATCH_DATE DATE ,<br /> PRIMARY KEY (COD_MATCH, MATCH_DATE)<br />);</code></pre></blockquote><p></p><blockquote><pre><code>CREATE TABLE USERS ( <br /> COD_MATCH INTEGER,<br /> FIRST_NAME VARCHAR(25),<br /> LAST_NAME VARCHAR(50),<br /> MATCH_DATE DATE,<br /> CONSTRAINT Match_Cod_Date_FK FOREIGN KEY (COD_MATCH, MATCH_DATE) <br / > REFERENCES MATCHES<br />);</code></pre></blockquote><p>Se tentarmos eliminar a tabela <code>MATCHES</code> com o comando</p ><blockquote><pre> <code>DROP TABLE MATCHES;</code></pre></blockquote><p>o seguinte erro seria retornado:</p><blockquote><p><code>ERROR : não é possível descartar correspondências de tabela porque outros objetos dependem dela</code></p></blockquote><p>O erro se deve ao fato de que na tabela <code>USERS</code> criamos uma key que faz referência à tabela <code>MATCHES</code>.</p><p>Para evitar este erro, vamos indicar que, em primeiro lugar, todas as restrições de chave estrangeira que foram criadas na tabela que queremos são apagados apagar. Para fazer isso, adicionaremos a cláusula <code>CASCADE</code> à instrução drop da tabela:</p><blockquote><pre><code>DROP TABLE MATCHES CASCADE;</code></pre>< /blockquote><h3>Exercício:</h3><p>Imagine que, no banco de dados, temos as seguintes duas tabelas criadas:</p><blockquote><pre><code>CREATE TABLE FACULTIES (<br / > COD_FACULTY CHAR(3) PRIMARY KEY,<br /> NAME VARCHAR(100)<br /> );<br /><br />CREATE TABLE MODULES (<br /> CODE INTEGER PRIMARY KEY,<br /> NAME VARCHAR(100),<br /> FACULTY CHAR(3) REFERENCES FACULTIES(COD_FACULTY)<br />);</code></pre></blockquote><p>Como você excluiria a tabela <code>FACULTIES </code> sem excluir a tabela <code>MODULES</code>?</p>
