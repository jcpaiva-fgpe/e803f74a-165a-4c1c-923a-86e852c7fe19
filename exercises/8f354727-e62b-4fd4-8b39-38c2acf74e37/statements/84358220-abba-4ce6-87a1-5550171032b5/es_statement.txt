<p>Hemos dicho anteriormente. que para eliminar una tabla utilizaremos el comando <code>DROP TABLE</code>.</p><h3>Sintaxis:</h3><blockquote><pre><code>DROP TABLE TableName [CASCADE];</code></pre></blockquote><p>La opci&oacute;n <code>CASCADE</code> se puede incluir para los casos en que alguna de las columnas sea clave ajena en otra tabla secundaria, lo que impedir&iacute;a su borrado. Al colocar esta opci&oacute;n las restricciones donde es clave ajena se borrar&aacute;n antes y a continuaci&oacute;n se eliminar&aacute; la tabla en cuesti&oacute;n.</p><h3>Ejemplo:</h3><p>Imaginemos que hemos creado dos tablas relacionadoas, como <code>MATCHES</code> y <code>USERS</code>, con las siguientes sentencias:</p><blockquote><pre><code>CREATE TABLE MATCHES (<br />    COD_MATCH INTEGER,<br />    MATCH_DATE DATE,<br />        PRIMARY KEY (COD_MATCH, MATCH_DATE)<br />);</code></pre></blockquote><p></p><blockquote><pre><code>CREATE TABLE USERS (<br />    COD_MATCH INTEGER,<br />    FIRST_NAME VARCHAR(25),<br />    LAST_NAME VARCHAR(50),<br />    MATCH_DATE DATE,<br />        CONSTRAINT Match_Cod_Date_FK FOREIGN KEY (COD_MATCH, MATCH_DATE)<br />        REFERENCES MATCHES<br />);</code></pre></blockquote><p>Si intent&aacute;ramos eliminar la tabla <code>MATCHES</code> con el comando</p><blockquote><pre><code>DROP TABLE MATCHES;</code></pre></blockquote><p>se nos devolver&iacute;a el siguiente error:</p><blockquote><p><code>ERROR:  cannot drop table matches because other objects depend on it</code></p></blockquote><p>El error se debe a que en la tabla <code>USERS</code> hemos creado una clave ajena que referencia a la tabla <code>MATCHES</code>.</p><p>Para evitar este error, vamos a indicar que, en primer lugar, se borren todas las restricciones de clave ajena que se hayan creado sobre la tabla que queremos borrar. Para ello, a&ntilde;adiremos la cl&aacute;usula <code>CASCADE</code> a la sentencia de borrado de la tabla:</p><blockquote><pre><code>DROP TABLE MATCHES CASCADE;</code></pre></blockquote><h3>Ejercicio:</h3><p>Imagina que, en la base de datos, tenemos las dos siguientes tablas creadas:</p><blockquote><pre><code>CREATE TABLE FACULTIES (<br />     COD_FACULTY CHAR(3) PRIMARY KEY,<br />     NAME VARCHAR(100)<br /> );<br /><br />CREATE TABLE MODULES (<br />    CODE INTEGER PRIMARY KEY,<br />    NAME VARCHAR(100),<br />    FACULTY CHAR(3) REFERENCES FACULTIES(COD_FACULTY)<br />);</code></pre></blockquote><p>&iquest;C&oacute;mo borrar&iacute;as la tabla <code>FACULTIES</code> sin borrar la tabla <code>MODULES</code>?</p>
