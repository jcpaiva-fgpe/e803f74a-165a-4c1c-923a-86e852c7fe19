<h2>Constraint <a href="https://www.postgresql.org/docs/14/ddl-constraints.html#DDL-CONSTRAINTS-PRIMARY-KEYS" target="_blank"><code>PRIMARY KEY</ code></a>.</h2><p>In the relational model, tables must have a primary key. It is evident that when we create the table we will have to indicate which columns will be part of that primary key.</p><p>The primary key makes the fields that form it <code>NOT NULL</code> and the values of the fields are of type <code>UNIQUE</code>.</p><p>Let's see how the <code>USERS</code> table would look if the primary key was the <code>Login</code> field :</p><blockquote><pre><code>CREATE TABLE USERS (Login VARCHAR (25) PRIMARY KEY);</code></pre></blockquote><p>Or by giving the constraint a name :</p><blockquote><pre><code>CREATE TABLE USERS (<br /> Login VARCHAR (25)<br /> CONSTRAINT Users_log_PK PRIMARY KEY<br />);</code></pre>< /blockquote><h3>Exercise:</h3><p>Identifies the <code>ID</code> attribute as the primary key of the table from the previous exercise, adding the corresponding restriction.</p>
