<p>During our learning we will have to create many tables, for this we will need to handle the data types that postgreSQL uses. In the following link you have a list of these types and their description.</p><p><a href="https://www.postgresql.org/docs/14/datatype.html" target="_blank"> Data types in PostgresSQL.</a></p><h3>Examples:</h3><p>Without trying to be too exhaustive, here are some examples.</p><ul><li>la height of a person:<ul><li>if we measure it in centimeters (185): it will be an integer, so it will have a <a href="https://www.postgresql.org/docs/14/ datatype-numeric.html#DATATYPE-INT" target="_blank"><code>INTEGER</code></a></li><li>if we measure it in meters (1.85): it will be a number with one digit in the integer part and 2 decimal digits, so it will correspond to a <a href="https://www.postgresql.org/docs/14/datatype-numeric.html#DATATYPE-NUMERIC-DECIMAL" target="_blank "><code>NUMERIC(3, 2)</code></a>.</li></ul></li><li>An <a href="https://www.postgresql.org /docs/14/datatype-datetime.html" target="_blank">date:</a><ul><li>If we are only interested in storing the day and not the time (06/21/2022) a <code>DATE</code> type will correspond.</li><li>If we are interested in storing the time (06/21/2022 13:30) it will correspond a <code>TIMESTAMP</code></li></ul></li><li>If the value to store does not correspond to the previous types, such as a passport (P12345678A) or a group (1A) we will associate the type <code>VARCHAR()</code>, even if it contains digits</li></ul><h3>Exercise:</h3><p>Let's create a table of <code>students</code> more complete than the view in the previous example, knowing that the student data is stored in the following attributes:</p><ul><li><code>ID</code>: which contains numbers of up to 8 digits without decimals.</li><li><code>NIF</code>: which contains a combination of numbers and letters of up to 9 characters.</li><li><code>LAST_NAME</code>: previously already we have limited the length of this attribute to 50 characters.</li><li><code>FIRST_NAME</code>: We have previously limited the length of this attribute to 25 characters.</ li><li><code>BIRTH_DATE</code>: Logically, it stores a date.</li><li><code>NATIONALITY</code>: It will store character strings with a maximum length of 30.</li ><li><code>EMAIL</code>: will also store strings of up to 100 characters.</li><li><code>PHONE</code>: will store only the digits, without spaces or prefix.</li ></ul>
